## 端口映射说明

### 后端端口映射
```yaml
ports:
  - "8880:8000"
  # 宿主机8880端口 -> 容器内8000端口
```

- **容器内端口**: 8000（应用监听端口，不变）
- **宿主机端口**: 8880（外部访问端口，已修改）

### 前端端口映射
```yaml
ports:
  - "18880:80"
  # 宿主机18880端口 -> 容器内80端口
```

- **容器内端口**: 80（Nginx 监听端口，不变）
- **宿主机端口**: 18880（外部访问端口，已修改）

## 修改端口的原因

原端口 80 和 8000 已被占用，因此修改为：
- 80 → 18880
- 8000 → 8880

## 如何修改端口

如果需要再次修改端口，编辑 `docker-compose.cn.yml`：

```yaml
services:
  backend:
    ports:
      - "新端口:8000"  # 修改宿主机端口
  
  frontend:
    ports:
      - "新端口:80"    # 修改宿主机端口
```

然后重新部署：
```bash
docker-compose -f docker-compose.cn.yml down
docker-compose -f docker-compose.cn.yml up -d
```

## 防火墙配置

如果需要外网访问，需要开放端口：

```bash
# CentOS 7 使用 firewalld
sudo firewall-cmd --permanent --add-port=8880/tcp
sudo firewall-cmd --permanent --add-port=18880/tcp
sudo firewall-cmd --reload

# 或使用 iptables
sudo iptables -I INPUT -p tcp --dport 8880 -j ACCEPT
sudo iptables -I INPUT -p tcp --dport 18880 -j ACCEPT
sudo service iptables save
```

## 验证端口

```bash
# 检查端口是否监听
netstat -tulpn | grep 8880
netstat -tulpn | grep 18880

# 或使用 ss 命令
ss -tulpn | grep 8880
ss -tulpn | grep 18880

# 测试端口连通性
curl http://localhost:8880/health
curl http://localhost:18880/
```

## 常见问题

### Q1: 端口仍然被占用怎么办？

```bash
# 查看占用端口的进程
sudo lsof -i :8880
sudo lsof -i :18880

# 或使用 netstat
sudo netstat -tulpn | grep 8880
```

### Q2: 修改端口后前端无法访问后端？

检查 CORS 配置是否包含新端口：
```env
CORS_ORIGINS=http://localhost:18880,http://服务器IP:18880
```

### Q3: 容器启动但无法访问？

```bash
# 检查容器状态
docker-compose -f docker-compose.cn.yml ps

# 查看容器日志
docker-compose -f docker-compose.cn.yml logs backend
docker-compose -f docker-compose.cn.yml logs frontend

# 检查容器内端口
docker-compose -f docker-compose.cn.yml exec backend netstat -tulpn
```
