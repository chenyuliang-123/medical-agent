# 部署脚本使用说明

## 脚本概览

项目提供了多个部署脚本，适用于不同场景：

| 脚本 | 用途 | 速度 | 使用场景 |
|------|------|------|----------|
| `quick-update.sh` | 快速更新 | ⚡ 1分钟 | 只修改了代码，依赖未变 |
| `rebuild.sh` | 重新构建 | 🔨 5-10分钟 | 修改了依赖（requirements.txt） |
| `deploy-cn.sh` | 完整部署 | 🐢 20-30分钟 | 首次部署或完全重建 |

## 使用指南

### 1. 快速更新（推荐日常使用）

**适用场景**：
- 修改了 Python 代码
- 修改了前端代码
- 修改了配置文件（.env）
- **未修改** requirements.txt

**执行命令**：
```bash
./quick-update.sh
```

**速度**：约 1 分钟

**原理**：只重启容器，不重新构建镜像

---

### 2. 重新构建

**适用场景**：
- 修改了 `requirements.txt`（添加/删除依赖）
- 修改了 `package.json`（前端依赖）
- 修改了 Dockerfile

**执行命令**：
```bash
./rebuild.sh
```

**速度**：5-10 分钟（使用 Docker 缓存）

**原理**：重新构建镜像，但利用 Docker 层缓存

---

### 3. 完整部署

**适用场景**：
- 首次部署
- 需要完全清理重建
- 遇到缓存问题

**执行命令**：
```bash
./deploy-cn.sh
```

**速度**：20-30 分钟

**原理**：完整构建流程，包括环境检查

---

## 常见操作

### 只修改了后端代码

```bash
# 方法1: 快速更新（推荐）
./quick-update.sh

# 方法2: 只重启后端
docker-compose -f docker-compose.cn.yml restart backend
docker-compose -f docker-compose.cn.yml logs -f backend
```

### 只修改了前端代码

```bash
# 方法1: 快速更新
./quick-update.sh

# 方法2: 只重启前端
docker-compose -f docker-compose.cn.yml restart frontend
```

### 修改了 requirements.txt

```bash
# 必须重新构建
./rebuild.sh
```

### 修改了 .env 配置

```bash
# 快速更新即可
./quick-update.sh
```

### 查看日志

```bash
# 查看所有服务日志
docker-compose -f docker-compose.cn.yml logs -f

# 只看后端日志
docker-compose -f docker-compose.cn.yml logs -f backend

# 只看前端日志
docker-compose -f docker-compose.cn.yml logs -f frontend

# 查看最近50行
docker-compose -f docker-compose.cn.yml logs --tail=50
```

### 停止服务

```bash
docker-compose -f docker-compose.cn.yml down
```

### 重启服务

```bash
# 重启所有服务
docker-compose -f docker-compose.cn.yml restart

# 只重启后端
docker-compose -f docker-compose.cn.yml restart backend

# 只重启前端
docker-compose -f docker-compose.cn.yml restart frontend
```

### 查看服务状态

```bash
docker-compose -f docker-compose.cn.yml ps
```

## 故障排查

### 服务启动失败

```bash
# 1. 查看日志
docker-compose -f docker-compose.cn.yml logs backend

# 2. 进入容器调试
docker exec -it agent-backend bash

# 3. 检查配置
docker exec agent-backend env | grep DATABASE_URL
```

### 端口被占用

```bash
# 查看端口占用
sudo lsof -i :9880
sudo lsof -i :19880

# 停止占用进程
sudo kill -9 <PID>
```

### 构建太慢

```bash
# 不要使用 deploy-cn.sh，使用 rebuild.sh
./rebuild.sh

# 或者直接使用 docker-compose
docker-compose -f docker-compose.cn.yml build
docker-compose -f docker-compose.cn.yml up -d
```

## 最佳实践

### 日常开发流程

1. **修改代码**
2. **提交到 Git**
3. **上传到服务器**
   ```bash
   scp -r ./backend root@10.10.31.193:/opt/ai_slow_hj/agent/
   ```
4. **快速更新**
   ```bash
   ssh root@10.10.31.193
   cd /opt/ai_slow_hj/agent
   ./quick-update.sh
   ```

### 添加新依赖

1. **修改 requirements.txt**
2. **上传到服务器**
3. **重新构建**
   ```bash
   ./rebuild.sh
   ```

### 首次部署

1. **上传所有文件**
2. **配置 .env**
3. **完整部署**
   ```bash
   ./deploy-cn.sh
   ```

| 配置更新 | 20-30 分钟 | 1 分钟 | 95% ⚡ |

## 总结

- **日常使用**：`quick-update.sh` ⚡
- **依赖变化**：`rebuild.sh` 🔨
- **首次部署**：`deploy-cn.sh` 🚀

**记住：大部分情况下使用 `quick-update.sh` 即可！**
